# ADR-018: Docs Page for Minor/Major Releases

## Status

Accepted (2026-03-04)

## Context

Minor and major releases are rare events that require manual steps (set version, create git tag, push tag, optionally create GitHub Release). This process could be automated via a dedicated `/opsx:release` skill or documented in a docs page.

## Decision

Document the manual minor/major release process in docs — rare enough for a manual process.

## Rationale

Rare enough that a docs page suffices. A dedicated skill would be over-engineering for current needs.

## Alternatives Considered

- Dedicated `/opsx:release` skill (over-engineering for current needs)

## Consequences

- No `/opsx:release` skill — deferred for the future.
- Manual process documented and auto-generated by `/opsx:docs` from the release-workflow spec.
