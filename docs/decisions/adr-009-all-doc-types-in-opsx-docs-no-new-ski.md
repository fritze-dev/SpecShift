# ADR-009: All Doc Types in /opsx:docs, No New Skills

## Status

Accepted (2026-03-04)

## Context

The documentation ecosystem expansion (enriched capability docs, architecture overview, ADRs) could be delivered via separate skills or consolidated into the existing `/opsx:docs` command. The user preferred a single entry point.

## Decision

All documentation types generated by `/opsx:docs` — no separate `/opsx:adr` or `/opsx:research-log` skills.

## Rationale

User preference, avoids skill proliferation, single entry point.

## Alternatives Considered

- Separate `/opsx:adr` + `/opsx:research-log` skills (rejected)

## Consequences

- Single command generates all documentation — simpler for users.
- SKILL.md is more complex but well-structured with clear section headers.
