No critical gaps.

- **`.claude/skills/specshift/`**: legacy compiled tree. After this change, `compile-skills.sh` removes it. Currently committed to Git per CONSTITUTION ("It is committed to Git and MUST NOT be hand-edited"). After migration, the new `./skills/specshift/` is committed instead, and `.claude/skills/` is deleted from the working tree. `git status` will show the deletion as part of the change. **Verification:** include `.claude/skills/` deletion in the apply commit.
- **`.claude-plugin/marketplace.json`**: source path change. Existing Claude Code installs need a `marketplace update` to refresh. Already documented as the standard update flow.
- **GitHub Action that auto-tags on version push**: watches `src/.claude-plugin/plugin.json` (per CHANGELOG history). Unaffected by output-path changes. **Verification:** check `.github/workflows/` after compile to confirm.
- **`.specshift/WORKFLOW.md`**: synced from updated `src/templates/workflow.md`. Project's worktree config and review config remain.
- **`docs/capabilities/`**: a new capability doc for `multi-target-distribution` and updated `project-init.md` capability doc will be regenerated by `specshift finalize`.
- **README**: install section restructure visible to all users browsing the repo. Existing consumers see new section; no functional change for them.

No silent regressions identified.
