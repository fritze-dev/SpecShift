# Research: Codex Plugin Support

## 1. Current State

SpecShift is currently packaged for Claude Code. The source files live in `src/`, and `scripts/compile-skills.sh` generates the Claude release under `.claude/`. The generated Claude release contains `.claude/.claude-plugin/plugin.json`, `skills/specshift/SKILL.md`, copied templates, and compiled action requirement files.

Codex plugins use a similar skill layout and, following the Shopify AI Toolkit reference implementation, can be exposed directly from the repository root with `.codex-plugin/plugin.json` and a root-level `skills/` directory. Codex project-level instructions conventionally use `AGENTS.md`, while SpecShift currently bootstraps `CLAUDE.md`.

## 2. Existing Specs Reviewed

- `release-workflow`: owns source/release layout, marketplace source configuration, and `scripts/compile-skills.sh`.
- `project-init`: owns bootstrap behavior and the agent instruction file generated by `specshift init`.
- `three-layer-architecture`: owns the router/skill architecture and proactive trigger rules.
- `workflow-contract`: owns runtime action dispatch and compiled requirement loading.

## 3. Findings

- The safest implementation keeps `src/` as the source of truth and extends the compiler to generate additional Codex root-plugin artifacts.
- Shopify AI Toolkit demonstrates the preferred Codex layout for this kind of public plugin repository: `.codex-plugin/plugin.json` and `skills/` live directly at the repository root, not inside a nested `plugins/<name>/` directory.
- Codex-specific output can be produced by deterministic transformations from existing source: `CLAUDE.md` -> `AGENTS.md`, `.claude-plugin` -> `.codex-plugin`, and `.claude/worktrees` -> `.specshift/worktrees`.
- A small Codex-specific manifest source is still needed because Codex plugin manifests require an `interface` block and have different marketplace metadata.
- The Codex skill trigger should be less aggressive than the Claude trigger because Codex has broad coding-task skill discovery and should not route every normal implementation request through SpecShift unless the repository is already governed by `.specshift`.

## 4. Clarification Assessment

| Category | Status | Notes |

| Scope | Clear | Add installable Codex plugin support while preserving Claude support. |

## 5. Decisions

- Generate Codex release artifacts at the repository root (`.codex-plugin/` and `skills/specshift/`) from `src/`, matching Shopify AI Toolkit.
- Add `src/.codex-plugin/plugin.json` as the Codex manifest source, copied to root `.codex-plugin/plugin.json` during compilation.
- Add `src/codex/templates/agents.md` for the Codex bootstrap template, because the file name and platform terminology differ from Claude.
- Keep the compiler in Bash with POSIX utilities, matching the existing no-runtime-dependencies constraint.
