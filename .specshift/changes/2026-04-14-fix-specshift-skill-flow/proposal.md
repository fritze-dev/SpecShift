## Why

Claude Code Web ignores the CLAUDE.md instruction to use the specshift skill flow when transitioning from plan mode to implementation. The skill's `description` field is passive (describes what, not when), so Claude never proactively invokes it. The CLAUDE.md enforcement text excludes source code files from its "never edit directly" rule.

## What Changes

- Add TRIGGER/DO NOT TRIGGER conditions to the specshift skill `description` field in `src/skills/specshift/SKILL.md`, using the proven pattern from other Claude Code skills (e.g., `claude-api`)
- Strengthen the CLAUDE.md workflow enforcement text to cover ALL file types and explicitly name the skill invocations as gates
- Sync the same enforcement text to the consumer CLAUDE.md template (`src/templates/claude.md`) with template-version bump
