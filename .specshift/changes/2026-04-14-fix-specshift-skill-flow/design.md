# Technical Design: Fix SpecShift Skill Flow Triggering

## Context

Claude Code Web ignores the CLAUDE.md workflow enforcement after plan mode exit. Two instruction-level gaps allow the AI to bypass the specshift skill and edit files directly. The fix is straightforward — two text changes to existing files with no architectural implications.

## Architecture & Components

- `src/skills/specshift/SKILL.md` (line 3) — `description` frontmatter field. Controls the skill trigger hint in the system prompt.
- `CLAUDE.md` (line 5) — Workflow enforcement text. Loaded as project instructions in every session.
- `src/templates/claude.md` (line 17) — Consumer template. Must match CLAUDE.md enforcement text.
- `.claude/skills/specshift/SKILL.md` — Compiled release. Auto-regenerated by `compile-skills.sh`.

## Goals & Success Metrics

- SKILL.md description includes TRIGGER/DO NOT TRIGGER conditions — PASS/FAIL by inspection
- CLAUDE.md says "Before editing ANY file" and names specshift skill explicitly — PASS/FAIL by inspection
- `compile-skills.sh` runs without errors — PASS/FAIL
- Consumer template version bumped to 2 — PASS/FAIL by inspection

## Non-Goals

- Runtime hook enforcement (`.claude/settings.json`)
- WORKFLOW.md action instruction changes
- Changes to the pipeline or artifact structure

## Decisions

| Decision | Rationale | Alternatives |
|----------|-----------|--------------|
| Use TRIGGER/DO NOT TRIGGER pattern in description | Proven pattern from claude-api skill; directly addresses root cause | Hook-based enforcement (rejected — unreliable, removed previously) |
| Cover "ANY file" in CLAUDE.md text | Current text excludes source code, creating a loophole | List all file types exhaustively (verbose, fragile) |
| Skip hooks in .claude/settings.json | Previous SessionStart hook removed for unreliability (commit 97da853) | Could be added later if instruction fix proves insufficient |

## Risks & Trade-offs

- [Over-triggering] → Mitigated by DO NOT TRIGGER conditions covering read-only activities
- [Description length increase] → Negligible token impact (~200 extra chars in system prompt)
- [Consumer template version bump] → Consumers get update on next `specshift init` (intended upgrade path)
