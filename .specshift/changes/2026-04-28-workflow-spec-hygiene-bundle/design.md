# Technical Design: Workflow/Spec-Hygiene Bundle (#59 + #58)

## Context

SpecShift's workflow templates and several specs use positional structural identifiers — numerical (`## 1. Foundation`) and alphabetic (`## A. Traceability Matrix`) — for top-level sections in artifact templates, plus hardcoded "8-stage pipeline" counts in three locations. Cross-references like "section 4 with universal post-implementation steps" rot silently when sections move; the `artifact-pipeline.md` spec on Line 328 even acknowledges this with a hedge clause.

Separately, the `## Action: review` Self-check bullet in `src/templates/workflow.md` is one of eleven bullets, with no marker, no audit hook, and no Pre-merge gate. The Self-Check was caught only by user reminder during PR #50.

Both touch the same workflow surface (`src/templates/workflow.md`). Bundling them gives a single template-version-bump cascade, one consumer merge prompt, and one audit pass. Specs pre-edited in this propose phase capture both the heading-discipline convention (new requirement in `artifact-pipeline.md`) and the self-check enforcement (new requirement in `review-lifecycle.md`).

## Architecture & Components

**Templates layer (`src/templates/`):**
- `workflow.md` — `## Action: finalize` `### Instruction` numbered list → semantic bullets. `## Action: review` `### Instruction` Self-check bullet sharpened (HOW + marker), Pre-merge summary bullet gated.
- `changes/tasks.md` — five `## N.`-prefixed sections → semantic; instruction-block self-references named.
- `changes/research.md` — seven `## N.`-prefixed sections → semantic.
- `changes/preflight.md` — seven `## A.`–`## G.`-prefixed sections → semantic.
- `changes/audit.md` — instruction wording: dimensions named, not numbered.
- All five templates: `template-version` bumped (4→5, 1→2, 1→2, 2→3, 11→12).

**Specs layer (`docs/specs/`):**
- `artifact-pipeline.md` — Purpose line, Eight-Stage Pipeline requirement renamed (Pipeline Stages and Dependencies) and dropped count, Standard Tasks Directive scenarios named, Edge Case "Custom section numbering" removed, NEW requirement "Semantic Heading Structure in Pipeline Artifact Templates". Version 6→7.
- `three-layer-architecture.md` — "7-stage" / "exactly 8 artifact IDs" → cross-spec normative reference. Version 6→7.
- `review-lifecycle.md` — Review Comment Processing wording made normative-reference; NEW requirement "Self-Check Mandatory After Comment Processing" with four scenarios; Pre-Merge Summary Comment requirement gated. Version 4→5.

**Manifests / docs:**
- `src/actions/review.md` — link list extended to include the new requirement.
- `README.md:64` — "**8-stage pipeline**" → "**artifact pipeline**".

**Mirrors (synced from `src/templates/`):**
- `.specshift/WORKFLOW.md` — hand-edited; mirror finalize bullets and review action sharpening; preserve project-specific `Compile` step.
- `.specshift/templates/changes/{tasks,research,preflight,audit}.md` — re-sync from `src/templates/`.

**Generated (not edited directly):**
- `skills/specshift/**` — regenerated by `bash scripts/compile-skills.sh` during finalize.
- `docs/capabilities/{artifact-pipeline,three-layer-architecture,review-lifecycle}.md` — regenerated during finalize.

**Component interactions:**
- `scripts/compile-skills.sh` reads `src/templates/**` and `src/actions/**`, validates `template-version` bumps against `main`, regenerates `skills/specshift/**`. Will fail the build if any of the five bumps is missing.
- `specshift apply` iterates `tasks.md`, edits the listed files, regenerates `skills/specshift/**`, runs `audit.md` against the change scope.
- `specshift review` (after merge of this change) will produce a `<!-- specshift:self-check -->` PR comment as part of its own lifecycle — the change validates itself in production on first reuse.

## Non-Goals

- Does NOT change router behavior (`src/skills/specshift/SKILL.md` not edited).
- Does NOT change YAML `pipeline:` array order or content (already semantic).
- Does NOT change `actions:` array order (sequence-by-design).
- Does NOT add audit/preflight artifact to `review` action (marker comment is the chosen mechanism, not a local file).
- Does NOT renumber historical artifacts under `.specshift/changes/<date>-<slug>/`.
- Does NOT touch CHANGELOG.md historical references (`step 3`, `step 7`, `step 8` from past versions on Lines 16, 149, 174).
- Does NOT fix Issue #57 HTML-comment-bug (separate change with side-effect investigation).
- Does NOT bump `template-version` on untouched templates (`design.md`, `tests.md`, `proposal.md`, `claude.md`, `agents.md`, `constitution.md`, `docs/*.md`).

## Decisions

| Decision | Rationale | Alternatives |
|----------|-----------|--------------|
| Bundle #59 + #58 in one change | Both touch `src/templates/workflow.md`; single template-version cascade; one consumer merge prompt; one audit pass | Two sequential changes (rejected: rebase pain on shared file) |
| Drop both numeric (`## 1.`) and alphabetic (`## A.`) prefixes uniformly | Matches existing spec pattern (`### Requirement: <Name>`); zero parser dependencies in `src/skills/specshift/SKILL.md` | Keep prefixes with anchors (rejected: no external linkers); fix only headings, leave self-references (rejected: still rots) |
| Resolve hardcoded "8-stage" / "exactly 8" / "7-stage" | User pushback during planning ("sollen wir die harte Nummerierung auf 8 nicht direkt auflösen?"); count drift between Lines 30 and 37 of three-layer-architecture.md proves the rot | Leave the count (rejected: actual drift already happened) |
| Self-Check marker as PR comment `<!-- specshift:self-check -->` | HEAD-commit-anchored; re-entrant; pre-merge gate uses identical inspection pattern as existing `<!-- specshift:review-summary -->` | Local `self-check.md` artifact (rejected: stale across HEAD updates, not reviewer-visible); router auto-dispatch (rejected: too invasive per Issue #58 OOS) |
| Pre-merge summary refuses to post when no marker exists for current HEAD | Makes the omission impossible to ship past the merge gate; observable failure mode | Make self-check advisory (rejected: status quo, doesn't fix #58); add audit.md-style verdict (rejected: extra artifact for a single boolean) |
| Replace advisory wording in existing "Review Comment Processing" requirement with normative reference | Avoids duplicate "self-check should run" sentences; new requirement carries the enforcement contract | Add new requirement alongside existing advisory text (rejected: drift risk between two specs of the same rule) |
| Add new "Semantic Heading Structure in Pipeline Artifact Templates" requirement to `artifact-pipeline.md` | Codifies the convention for future template edits; provides a normative answer if a contributor asks "why no leading number" | Leave convention undocumented (rejected: future regress risk); add to spec-format.md (rejected: spec-format covers spec files, not change templates) |
| Issue #57 stays out of scope | HTML-comment side-effect investigation explicitly scoped to a separate change; preflight.md is touched here only at headings | Fold #57 in (rejected: would expand audit scope and delay this bundle) |

## Risks & Trade-offs

- **Five template-version bumps in one change → one consumer merge prompt** → Mitigation: Bundle is the lowest-friction path (single prompt vs. two prompts in sequential changes); CHANGELOG entry will document the cosmetic nature.
- **`.specshift/WORKFLOW.md` divergence from `src/templates/workflow.md`** (project mirror has extra `Compile` step) → Mitigation: Edit both files in same task; preserve `Compile` step in mirror; verify by reading both after edit.
- **Self-check marker pattern depends on GitHub tooling** (PR comment must be writable) → Mitigation: Existing `<!-- specshift:review-summary -->` already depends on identical capability; this assumption is shared.
- **New Semantic Heading requirement might surprise consumers** who customized templates with numeric prefixes → Mitigation: Requirement explicitly says "SHALL be removed when the template is next modified" (lazy enforcement); historical artifacts excluded.
- **Three-layer-architecture.md had silent drift** ("7-stage" Line 30 vs "exactly 8" Line 37) — this change fixes the drift but may surprise readers who took Line 30 at face value → Mitigation: design.md flags this as a known existing inconsistency that the change resolves.

## Migration Plan

- **Apply phase:** Edit all listed files in tasks.md order; run `bash scripts/compile-skills.sh` to regenerate `skills/specshift/**`; verify Goals & Success Metrics greps return zero matches.
- **Finalize phase:** Update `CHANGELOG.md` with semantic-headings + self-check-enforcement entries (single block under one version); regenerate `docs/capabilities/`; bump `src/VERSION` per project convention.
- **Review phase:** This is the first change to use the new self-check marker — the review action will validate itself by producing the marker. If the marker pattern fails (e.g., GitHub tooling unavailable in this environment), fall back to advisory mode and document.
- **Rollback strategy:** Pure markdown/YAML edits — `git revert` of the squash merge restores prior templates and specs. No code, no migrations, no schema changes. Consumers who already merged the template-version bump will see no further prompts on revert (template-version monotonically increases).
