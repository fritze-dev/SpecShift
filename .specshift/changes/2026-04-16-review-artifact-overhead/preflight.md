## Duplication & Consistency

Two consistency findings in adjacent specs that are direct consequences of deleting `research.md` and `tests.md` templates:

1. **`docs/specs/project-init.md` lines 80-91** — Two scenarios use `research.md` as the illustrative template name in init's template-update behavior:
   - Scenario "Unchanged template updated silently"
   - Scenario "User-customized template preserved"
   With `research.md` deleted, these scenarios reference a non-existent template. The underlying requirement (template-version-based init merge) is unchanged; only the example needs swapping (e.g., to `proposal.md` or `audit.md`).
   **Severity: WARNING.**

2. **`docs/specs/change-workspace.md` line 157** — Scenario describes "a change at `.specshift/changes/2026-04-01-add-auth/` with research.md and proposal.md (status: active) but no tasks.md". The example file list mentions research.md. Not strictly wrong (legacy changes still have research.md), but a new change won't have research.md, so the scenario reads as if research.md is still part of the standard layout.
   **Severity: WARNING (low).** Could be left as-is for legacy-tolerance illustration, or updated to "with proposal.md (status: active) but no tasks.md".

Both findings are mechanical fixes (no behavior change). Recommendation: include in tasks.md as part of the same change, since the deletions cause them.
