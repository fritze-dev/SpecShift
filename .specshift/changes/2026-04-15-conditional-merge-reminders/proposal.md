## Why

Post-merge reminders in tasks.md are always included unconditionally, even for changes that don't touch plugin-distributed files. The "Update plugin locally" reminder is noise for docs-only or constitution-only changes. The template instruction should filter post-merge items by relevance to the change scope.

## What Changes

- Update the tasks.md Smart Template instruction to evaluate post-merge item relevance against the proposal's scope before including them
- Update the project constitution's post-merge item to include a scope hint indicating when it applies
- Update the consumer constitution template example to show scope-aware post-merge items
- Update specs to document conditional post-merge inclusion behavior
