## Why

The `specshift review` action processes review comments, runs self-checks, and asks for merge confirmation — but only reports results in the chat session. There is no audit trail on the PR itself, so anyone watching the PR on GitHub has no visibility into what the AI did before merge. This change adds a summary comment posted to the PR before merge confirmation. Closes #29.

## What Changes

- Add a new requirement "Pre-Merge Summary Comment" to the review lifecycle spec, defining the summary content (threads resolved, fixes applied, self-check result, cycles completed), graceful failure behavior, and re-entrant idempotency via an HTML marker.
- Add a new edge case (permissions denied) and assumption (PR issue comment read-write capability) to the spec.
- Convert the review action's numbered workflow steps to descriptive phase labels, eliminating fragile renumbering and making back-references clearer.
- Add a new "Pre-merge summary" phase between CI gate and merge confirmation in the workflow template.
